## ✅ Стабильный доступ к БД — Vercel API (РАБОТАЕТ)

**MCP Supabase — НЕ ИСПОЛЬЗОВАТЬ.** Нестабилен, требует OAuth.

```

```

### Проверка в начале каждой сессии:
```
GET https://tea-production-db.vercel.app/api/status
```
Ожидаемый ответ: `"ok": true`

### Все эндпоинты:

| Запрос | URL |
|---|---|
| Статус + счётчики | `GET /api/status` |
| Поиск сырья | `GET /api/materials?search=ЗАПРОС` |
| Артикул точно | `GET /api/materials?article=5214/1` |
| Все рецепты | `GET /api/recipes` |
| Рецепт по артикулу | `GET /api/recipes?article=2306` |
| Состав рецепта | `GET /api/recipes?id=ID&ingredients=1` |
| Все SKU | `GET /api/sku` |
| SKU по артикулу | `GET /api/sku?article=2306-ПА500` |
| Себестоимость SKU | `GET /api/sku?costs=1` |
| Себестоимость рецептов | `GET /api/sku?recipe_costs=1` |

## Схема БД

### Таблицы
- **raw_materials** — сырьё (444 позиции): `id, article, name, unit, category_id`
- **material_categories** — категории сырья
- **material_prices** — цены с историей: `id, material_id, price, valid_from`
- **recipes** — рецепты: `id, article, name, output_quantity, output_unit, notes, is_active`
- **recipe_ingredients** — состав: `id, recipe_id, material_id, sub_recipe_id, quantity, unit`
- **products** — SKU: `id, article, name, type_id, package_size, package_unit, recipe_id`
- **sku_recipe_components** — компоненты SKU: `id, product_id, recipe_id, material_id, quantity, unit`
- **price_lists** / **price_list_items** — прайслисты

### Views
- **recipe_cost** → `recipe_id, recipe_article, recipe_name, total_cost, cost_per_kg`
- **sku_cost** → `product_id, sku_article, product_name, blend_cost, packaging_cost, total_sku_cost`
- **product_pricing** → итоговое ценообразование

---

## Правила ввода данных

- Единицы: только **кг** и **шт** (граммы запрещены)
- Артикулы строго как указано, могут содержать `/`
- `output_quantity = 1.000`, `output_unit = 'кг'` для всех рецептов
- Цены в рублях, с датой `valid_from`
- Рецепт может ссылаться на другой через `sub_recipe_id`
